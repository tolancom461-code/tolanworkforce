# خطة تنفيذ نظام إشعارات دفعات الرواتب (Payroll Notifications)

بناءً على التحليل التقني لمشروع `TolanWorkforce` والمتطلبات التي ذكرتها، إليك الخطة المقترحة لتنفيذ نظام الإشعارات.

## 1. التغييرات في قاعدة البيانات (Schema)

سنقوم بإنشاء جدول جديد باسم `notifications` لتخزين الإشعارات.

| الحقل | النوع | الوصف |
|---|---|---|
| `id` | `int` | معرف فريد تلقائي |
| `userId` | `int` | معرف المستخدم المستلم للإشعار |
| `title` | `varchar` | عنوان الإشعار (مثلاً: اعتماد دفعة، رفض دفعة) |
| `message` | `text` | محتوى الإشعار بالتفصيل |
| `type` | `enum` | نوع الإشعار (`success`, `warning`, `info`, `error`) |
| `link` | `varchar` | رابط للانتقال السريع للدفعة المعنية |
| `isRead` | `boolean` | حالة القراءة (تلقائياً `false`) |
| `createdAt` | `timestamp` | تاريخ الإنشاء |

## 2. مسار العمل في الواجهة الخلفية (Backend Workflow)

سيتم تعديل الدوال التالية في `server/db.ts` لإطلاق الإشعارات تلقائياً عند تغيير حالة الدفعة:

### أ. عند اعتماد المدير المالي (`approveBatch`):
*   **المستلمون:** (مُعد الدفعة `createdBy` + المحاسب `accountantApprovedBy` + المراجع `auditorApprovedBy`).
*   **الرسالة:** "تم الاعتماد النهائي للدفعة [رقم الدفعة] من قبل المدير المالي."

### ب. عند رفض المدير المالي (`rejectBatch`):
*   **المستلمون:** (مُعد الدفعة `createdBy` + المحاسب `accountantApprovedBy` + المراجع `auditorApprovedBy`).
*   **الرسالة:** "تم رفض الدفعة [رقم الدفعة] من قبل المدير المالي. السبب: [سبب الرفض]."

### ج. مراحل إضافية (اختياري لضمان التكامل):
*   عند إرسال الشؤون الإدارية للمحاسب ← إشعار للمحاسبين.
*   عند اعتماد المحاسب ← إشعار للمراجعين.
*   عند اعتماد المراجع ← إشعار للمدير المالي.

## 3. الواجهة الأمامية (Frontend)

*   **أيقونة الإشعارات:** إضافة أيقونة "جرس" في الهيدر بجانب زر التثبيت والوضع الليلي.
*   **قائمة منسدلة (Dropdown):** تعرض آخر 5 إشعارات مع إمكانية تعليم الكل كمقروء.
*   **عداد الإشعارات:** دائرة حمراء صغيرة تظهر عدد الإشعارات غير المقروءة.
*   **تنبيهات لحظية (Toasts):** استخدام مكتبة `sonner` (الموجودة فعلياً في المشروع) لإظهار تنبيه منبثق فور وصول إشعار جديد أثناء تصفح البرنامج.

## 4. الخطوات القادمة بعد موافقتك

1.  تعديل ملف `schema.ts` وإجراء `db:push`.
2.  إضافة `notificationsRouter` في `server/routers.ts` للتعامل مع جلب وتحديث الإشعارات.
3.  تعديل دوال الوورك فلو في `server/db.ts`.
4.  بناء مكون `NotificationCenter` في الواجهة الأمامية.
