# تقرير تحليلي: مشكلة عدم ظهور الأجور اليومية في مسودة الراتب

**التاريخ:** 30 يناير 2026  
**المجموعة:** مؤسسة الملقا  
**المشكلة:** عند إنشاء مسودة راتب، لم تظهر الأجور اليومية وظهرت فقط الاستثناءات

---

## 🔴 تحديد المشكلة

عند إنشاء مسودة راتب لمجموعة مؤسسة الملقا:
- ❌ لم تظهر الأجور اليومية (baseAmount)
- ❌ لم تظهر الخصومات المحسوبة من الحضور والانصراف
- ✅ ظهرت فقط الاستثناءات (pay_overrides)

---

## 🔍 تحليل تدفق البيانات

### المسار الصحيح للأجور:

```
الحضور والانصراف (attendance_events)
        ↓
حساب الأجور اليومية (calculateDailyFinanceFromAttendance)
        ↓
السجلات المالية اليومية (worker_daily_finance)
        ↓
مسودة الراتب (payroll_batches)
```

### ما يحدث فعلياً:

```
مسودة الراتب (payroll_batches)
        ↓
البحث في (worker_daily_finance)
        ↓
إذا كانت فارغة → الأجور = 0
```

---

## 🎯 السبب الجذري

### **المشكلة الأساسية:**

في دالة `createPayrollBatch` (السطر 1876-1907 في db.ts)، النظام يبحث عن بيانات الأجور في جدول `worker_daily_finance`:

```typescript
// db.ts - السطور 1880-1889
const dailyRecords = await db
  .select()
  .from(workerDailyFinance)
  .where(
    and(
      eq(workerDailyFinance.workerId, worker.id),
      sql`${workerDailyFinance.workDate} >= ${params.periodStart}`,
      sql`${workerDailyFinance.workDate} <= ${params.periodEnd}`
    )
  );
```

**إذا لم تكن هناك سجلات في `worker_daily_finance`:**
- `daysWorked = 0`
- `baseAmount = 0`
- `totalDeductions = 0`
- `totalBonuses = 0`

---

## ❓ لماذا جدول `worker_daily_finance` فارغ؟

### السبب 1: **عدم معالجة الحضور والانصراف تلقائياً**

جدول `worker_daily_finance` يُملأ فقط عند:
1. استدعاء دالة `processAttendanceToFinance()` (السطر 1082)
2. أو استدعاء `createOrUpdateDailyFinance()` يدويًا

**المشكلة:** عند تسجيل الحضور والانصراف، لا يتم استدعاء هذه الدوال تلقائياً!

### السبب 2: **عدم وجود خط أنابيب معالجة**

النظام الحالي **لا يحتوي على:**
- ✗ معالج تلقائي لحساب الأجور اليومية
- ✗ دالة تُستدعى عند تسجيل الحضور
- ✗ خدمة خلفية تحسب الأجور بشكل دوري

### السبب 3: **الاعتماد على المستخدم**

النظام يتوقع من المستخدم أن يقوم بـ:
1. تسجيل الحضور والانصراف ✅
2. **ثم حساب الأجور اليومية يدويًا** ❌ (غير موجود)
3. **ثم إنشاء مسودة الراتب** ❌ (تجد جدول فارغ)

## 📊 تتبع البيانات

### مثال عملي:

**السيناريو:**
- عامل: أحمد محمد
- التاريخ: 2026-01-30
- الحضور: 08:15 (متأخر 15 دقيقة)
- الانصراف: 17:00

**ما يجب أن يحدث:**
```
1. تسجيل الحضور والانصراف في attendance_events ✅
2. حساب الأجور اليومية:
   - الأجر الأساسي: 100 ريال
   - الخصم من التأخير: 5 ريال
   - الصافي: 95 ريال
3. حفظ في worker_daily_finance ❌ (لا يحدث)
4. عند إنشاء مسودة الراتب:
   - البحث في worker_daily_finance
   - لا يجد سجلات
   - الأجر = 0 ❌
```

**ما يحدث فعلياً:**
```
1. تسجيل الحضور والانصراف ✅
2. ✗ لا يتم حساب الأجور اليومية
3. ✗ لا يتم حفظ البيانات
4. عند إنشاء مسودة الراتب:
   - البحث في worker_daily_finance (فارغ)
   - الأجر = 0 ❌
```

### 1. **النظام الحالي صحيح من الناحية المنطقية**
